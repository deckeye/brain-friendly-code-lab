## ✨ 特徴

### 4つの設計原則

```
⚡ エラーを起こさせない
   └─ 制約と補助機能で未然に防ぐ

🔍 エラーを早期発見
   └─ リアルタイムフィードバックで即座に指摘

🔄 エラーから迅速回復
   └─ 自動保存とUndoで安心を提供

🧠 認知負荷を最小化
   └─ 情報を記憶させない、見れば分かるUI
```

- ✅ **実サービス事例分析** - SmartDB、楽楽精算、Notionなど
- ✅ **完全なチェックリスト** - 実装漏れを防止

## 🧠 認知負荷軽減のポイント

このガイドラインは、以下のような方々でも快適に使えるUIを実現します：

- 🎯 **マルチタスク環境で作業する方** - 段階的な情報開示、コンテキスト情報の常時表示
- ⏰ **時間的制約がある方** - デフォルト値と自動入力、スマート補完機能
- 🔄 **作業が頻繁に中断される方** - 入力済みサマリーの表示、スムーズな復帰サポート
- 🎨 **集中力の維持が難しい方** - 視覚的な手がかり、小さなゴール設定
- 🧩 **短期記憶に不安がある方** - 情報を記憶させない設計、インラインヘルプ
